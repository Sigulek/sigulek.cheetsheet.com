HTML Document Structure

Main elements

Creating a document, we write down a ruleset for the machine to interpret. That means we have to follow rules to render the content properly. To mark a document html we have to declare it on the first line with <!DOCTYPE html>. The declaretion is followed by <html> element to indicate limits of the page. The page, in turn, is splitted on 2 main sections: head and body.

Head

The <head> block is devoted to information for the machine: fonts, characters, stylesheets, and other metadata. Therefore, this block is not rendered except for the title.

Body

The <body> block is dedicated to humans, hence it is more complicated. We can think of this block as a story. Good story has a navigation or starter, main part with all the details, and the end with farewell. These sections are called <header>, <main>, and <footer>.

Header

It's common practice to use header for a navigation bar or some introducing information. For navigation use the <nav> element.

Main

It is what it sounds like. Main part include everything you want to be displayed. I believe, you are well orginized, and understand how important it is to keep an order. For this purpose, html language has <div> element as a container for other elements.

Footer

This element is used to conclude the page with contacts, privacy policy, and things like that.
